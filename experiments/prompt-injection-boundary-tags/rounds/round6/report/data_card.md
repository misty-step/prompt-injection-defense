## Collection Method

- **Malicious tool calls** (54 unique): Hand-crafted from real-world attack patterns documented in security research (Willison 2025, Rehberger 2025). Categories: sensitive file reads, environment/credential exfiltration commands, secret-containing messages to untrusted recipients.
- **Benign tool calls** (36 unique): Modeled on realistic developer workflows: reading source code, running tests/builds, messaging trusted admin addresses.
- **Evaluation**: Each of the 90 unique calls was evaluated against all 4 policy configurations, producing 360 rows.

## Limitations

1. **Hand-crafted, not model-generated.** Tool calls were authored by researchers, not generated by actual LLM responses to injection payloads. Real malicious calls may be more diverse or creative.
2. **Three tool types only.** Modern agents use many more tools. The policy gate's effectiveness on other tool types is untested.
3. **English-only.** All tool call arguments are in English. Multilingual evasion is not tested.
4. **Static snapshot.** The corpus represents attack patterns as of February 2026. Attack techniques evolve.

## Intended Use

- Evaluating tool-call policy gate configurations
- Benchmarking new filtering approaches against the labeled corpus
- Reproducing Round 6 findings
